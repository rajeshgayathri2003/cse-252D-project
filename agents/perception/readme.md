# ProcTHOR Perception Agent

## Overview
The Perception Agent serves as the "eyes" of the multi-agent embodied AI system. Designed to operate within the AI2-THOR/ProcTHOR synthetic environment, its primary role is to bridge the gap between raw visual pixel data and the natural language reasoning required by Large Language Models (LLMs). 

It continuously processes RGB frames from the simulator, identifies objects, calculates their spatial positions, and translates this data into both a text-based prompt for the Planning Agent and a persistently saved dataset for offline training or memory retrieval.

## Core Technologies
The agent utilizes the Ultralytics `auto_annotate` pipeline to chain two state-of-the-art vision models:
* **Detection (YOLO11m):** Scans the environment to identify objects and draw initial bounding boxes.
* **Segmentation (SAM2-Base):** Uses the YOLO bounding boxes as prompts to generate pixel-perfect polygon masks of the objects, allowing the agent to calculate accurate depth and screen coverage.

## How It Works
At every step in the agent's execution loop, the Perception Agent performs the following pipeline:
1.  **Ingestion:** Receives the current 2D RGB frame from the ProcTHOR controller.
2.  **Annotation:** Runs YOLO and SAM2 to detect and segment objects.
3.  **Spatial Translation:** Converts normalized polygon coordinates into qualitative, human-readable spatial descriptors (e.g., "top-left", "bottom-center").
4.  **Scale Estimation:** Calculates the percentage of the frame the object occupies to give the LLM an approximation of proximity/scale.
5.  **Data Persistence:** Saves the raw image and the annotation data permanently to the disk.

## Outputs

The Perception Agent generates two distinct types of outputs simultaneously:

### 1. LLM-Ready Text Output (Runtime)
The `perceive()` function returns a formatted natural language string designed to be injected directly into the prompt of the Planning Agent (e.g., Claude, GPT-4, DeepSeek). 

**Example Output:**
> Visible Objects (Segmented):
> - potted plant located in the center-right, covering roughly 3.5% of the view.
> - television located in the top-center, covering roughly 12.0% of the view.

*(Note: If no objects are detected, it returns: "No distinct objects are visible in the current view.")*

### 2. File System Output (Persistent Data)
To support dataset generation or external memory retrieval, the agent creates a strictly paired dataset on your hard drive under a `saved_agent_data` master directory. 

For every step, it generates an isolated sub-folder containing:
* `frame.jpg`: The raw RGB image captured from the simulator.
* `frame.txt`: The corresponding label file. 

**Structure:**
> /saved_agent_data
>     /step_0
>         frame.jpg
>         frame.txt
>     /step_1
>         frame.jpg
>         frame.txt

**Inside `frame.txt`:**
The text file uses the standard YOLO segmentation format. Each line represents one object. The first integer is the YOLO Class ID (e.g., 58 for potted plant), followed by a dynamic list of normalized polygon coordinates `[0.0 to 1.0]` mapping the exact outline of the object generated by SAM2.
> <class-id> <x1> <y1> <x2> <y2> ... <xn> <yn>
